# 业务架构文档

## 项目概述

ProxyPool 是一个基于 Python 3.11+、FastAPI 和 Redis 构建的现代化代理池系统，提供自动抓取、验证和管理代理IP的能力。

## 核心业务功能

### 1. 代理获取服务
- **当前实现**: 通过 `/get` 接口返回最高分代理
- **支持格式**: JSON 和纯文本格式

### 2. 代理统计服务
- **当前实现**: 通过 `/stats` 接口展示代理池健康状况
- **统计指标**: 总数量、高分代理数量、健康状态

### 3. 代理列表服务
- **当前实现**: 通过 `/all` 接口返回所有代理

## 过时的业务实现与改进计划

### 代理来源管理

#### 当前问题
- [ ] **代理源过少**: 仅支持 2 个免费代理源（kuaidaili、proxylistplus）
- [ ] **抓取策略单一**: 使用简单正则表达式，容易因网站结构变化而失效
- [ ] **无反爬虫机制**: 缺少 User-Agent 轮换和请求延迟，容易被封禁

#### 改进方案
- [ ] 扩展代理源至 10+ 个主流免费代理网站
- [ ] 实现模块化解析器接口，将"网络请求"与"HTML解析"解耦
- [ ] 添加 User-Agent 池和随机延迟机制
- [ ] 支持付费代理源接入（API 方式）
- [ ] 添加代理源健康度监控，自动禁用失效源

### 代理验证策略

#### 当前问题
- [ ] **单一验证目标**: 仅使用 httpbin.org 进行验证，存在单点故障风险
- [ ] **验证场景不匹配**: 不同业务场景需要不同的验证目标
- [ ] **验证频率固定**: 5分钟固定间隔，无法根据代理质量动态调整

#### 改进方案
- [ ] 支持多验证目标配置（Google、Bing、百度等）
- [ ] 实现按业务场景分类的验证策略
- [ ] 动态验证频率：高分代理降低验证频率，低分代理提高验证频率
- [ ] 添加验证目标健康检查，自动切换备用目标
- [ ] 支持自定义验证规则（响应时间、匿名性等）

### API 功能增强

#### 当前问题
- [ ] **无访问控制**: 任何人都可以无限制访问 API
- [ ] **过滤功能缺失**: 无法按协议（HTTP/HTTPS）、匿名性、地区筛选代理
- [ ] **无使用限制**: 缺少速率限制和配额管理
- [ ] **无可视化界面**: 仅提供 REST API，缺少 Web 管理界面

#### 改进方案
- [ ] 实现 API Key 认证机制
- [ ] 添加代理筛选参数（协议、匿名性、地区、响应时间）
- [ ] 实现速率限制（基于 IP 或 API Key）
- [ ] 开发 Web 管理面板（React/Vue）
  - [ ] 代理池实时监控
  - [ ] 分数分布可视化
  - [ ] 手动添加/删除代理
  - [ ] 代理源管理
- [ ] 添加 Webhook 通知（代理池告警）

### 评分机制优化

#### 当前问题
- [ ] **评分规则简单**: 仅基于成功/失败的固定加减分
- [ ] **无响应时间考量**: 不区分快速代理和慢速代理
- [ ] **无地域信息**: 无法根据地理位置优化代理选择

#### 改进方案
- [ ] 引入多维度评分体系
  - [ ] 成功率权重
  - [ ] 响应时间权重
  - [ ] 稳定性权重（连续成功次数）
- [ ] 记录代理响应时间，优先返回快速代理
- [ ] 添加地理位置信息（通过 IP 定位）
- [ ] 支持按业务场景的自定义评分策略

### 部署与运维

#### 当前问题
- [ ] **单机部署**: 无法水平扩展
- [ ] **无监控告警**: 缺少 Prometheus/Grafana 集成
- [ ] **日志分散**: 缺少集中式日志管理
- [ ] **无健康检查**: Docker 容器缺少健康检查端点

#### 改进方案
- [ ] 实现分布式架构
  - [ ] 抓取器独立部署
  - [ ] 验证器独立部署（可水平扩展）
  - [ ] API 服务独立部署
  - [ ] 使用 Redis Pub/Sub 或 Streams 进行组件通信
- [ ] 添加 `/health` 和 `/metrics` 端点
- [ ] 集成 Prometheus 指标导出
- [ ] 配置 Grafana 监控面板
- [ ] 实现集中式日志收集（ELK/Loki）
- [ ] 添加告警规则（代理池数量低于阈值、验证失败率过高等）
